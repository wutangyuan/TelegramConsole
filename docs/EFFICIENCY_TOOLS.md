# 效率工具使用说明

“效率工具”在 Telegram 登录成功后可用。它包含：消息搜索与操作、服务器定时、自动化规则、草稿与文件夹。

## 一、消息搜索与操作

### 搜索消息

1. 在左侧下拉框选择“全部会话”或某一个具体会话。
2. 输入消息关键词。
3. 点击“查询”。

结果会合并两个来源：

- `Telegram`：Telegram 服务器返回的历史搜索结果。
- `本地索引`：程序运行期间收到或加载过的消息，保存在 `messages.db`，并按 Telegram 账号隔离。

同一条消息同时出现在两个来源时只显示一条。本地索引不是 Telegram 全量历史备份；尚未加载且 Telegram 搜索未返回的旧消息不会凭空出现在本地结果中。

### Forum Topic

1. 先在会话下拉框选择一个具体的 Forum 群组或频道，不能选择“全部会话”。
2. 点击“加载话题”。
3. 选择 Topic 后再查询，结果只保留该 Topic 的消息。

普通群聊没有 Forum Topic，点击加载时 Telegram 会返回“不支持”或空列表。

### 消息操作

先在结果列表选中消息，再使用底部按钮：

- `回复`：在左下文本框填写正文，回复选中的消息。
- `引用回复`：填写正文，并把原消息正文作为引用发送。
- `编辑`：只能编辑当前账号自己发送且仍在 Telegram 可编辑期限内的消息。
- `删除`：支持多选；是否能删除他人消息取决于当前账号在群里的权限。
- `转发`：可多选消息，并在按钮左侧选择转发目标。
- `复制链接`：适用于公开频道、公开群或可生成内部链接的超级群；普通私聊和基础群没有公开消息链接。

这些操作最终是否成功仍受 Telegram 权限、消息类型和服务端限制控制。

## 二、服务器定时

服务器定时是 Telegram 原生的一次性计划消息。

### 创建任务

1. 选择目标私聊、群聊或频道。
2. 选择发送日期。
3. 输入本机时间，例如 `08:00`。
4. 输入消息正文。
5. 点击“创建”。

程序会立即把消息和发送时间提交给 Telegram。提交成功后由 Telegram 服务器计时，因此可以关闭本程序和电脑。

注意：

- 这是一次性任务，发送后不会自动生成下一次。
- 创建时间必须至少晚于当前时间 10 秒。
- “每天/每周签到”仍在主窗口的“定时签到”页，由 Quartz 执行，程序必须保持运行和登录。

### 查询与取消

选择目标会话后点击“刷新”，可以读取该会话尚未执行的服务器计划消息。选中一条或多条后点击“删除选中”即可取消。

## 三、自动化规则

自动化规则只在本程序运行、Telegram 在线并收到新消息时执行。它不是 Telegram 服务器托管任务。

### 字段说明

- `名称`：用于识别规则，例如“生产报警转发”。
- `触发条件`：决定如何匹配消息。
- `匹配内容`：关键词、正则表达式或发送人名称；部分触发条件不使用它。
- `限定会话`：可将任何规则限制在某个群或私聊；“全部会话”表示不限制。
- `动作`：仅记录日志、发送到 Telegram、发送邮件。
- `目标`：动作选择“发送到 Telegram”时使用。
- `邮件`：动作选择“发送邮件”时使用；SMTP 在主窗口“设置”中统一配置。
- `消息模板`：动作执行时发送或记录的正文。

### 触发条件

- `关键词`：消息正文包含匹配内容，不区分大小写。
- `正则表达式`：使用 .NET 正则表达式，不区分大小写；单次匹配最长 500 毫秒。
- `@我的消息`：Telegram 标记为提及当前账号，匹配内容可留空。
- `指定会话`：必须在“限定会话”选择具体会话，匹配内容可留空。
- `发送人`：发送人显示名称包含匹配内容，不区分大小写。

规则只处理别人发来的新消息，不处理当前账号自己发出的消息，因此规则通知不会再次触发自身形成循环。

### 动作

- `仅记录日志`：写入运行日志，不发送通知。
- `发送到 Telegram`：发送到“目标”选中的机器人、用户或群聊，并经过可靠发件箱防重复。
- `发送邮件`：发送到“邮件”中的地址，使用“设置 → 邮件 SMTP”的统一配置。

### 模板变量

中文和英文变量都可使用：

| 中文变量 | 英文变量 | 内容 |
| --- | --- | --- |
| `{规则}` | `{Rule}` | 规则名称 |
| `{群聊}` | `{Chat}` | 来源会话名称 |
| `{发送人}` | `{Sender}` | 发送人名称 |
| `{内容}` | `{Content}` | 原消息正文 |
| `{时间}` | `{Time}` | 消息时间 |

示例：

```text
规则 {规则} 命中：[{群聊}] {发送人}: {内容}
```

### 新增、修改与停用

- 新增：填写字段后点击“保存”。
- 修改：选中一条规则，点击“载入选中”，修改后再点击“保存”。
- 删除：可多选规则后点击“删除选中”。
- 启用/停用：双击规则列表中的一条记录。

规则配置保存在加密的 `settings.dat` 中，并按 Telegram 账号隔离。

### 示例

#### 例 1：群里出现“服务器宕机”时通知私聊

- 名称：`服务器报警`
- 触发条件：`关键词`
- 匹配内容：`服务器宕机`
- 限定会话：选择需要监控的工作群
- 动作：`发送到 Telegram`
- 目标：选择自己的私聊、机器人或通知群

#### 例 2：捕获错误码

- 触发条件：`正则表达式`
- 匹配内容：`\b(ERR|ERROR)-\d{3,6}\b`
- 动作：`发送邮件`
- 邮件：填写接收报警的邮箱

#### 例 3：记录所有 @ 我的消息

- 触发条件：`@我的消息`
- 匹配内容：留空
- 限定会话：`全部会话`
- 动作：`仅记录日志`

## 四、草稿与文件夹

### Telegram 云草稿

1. 选择会话。
2. 点击“读取云草稿”取得 Telegram 当前保存的草稿。
3. 修改正文后点击“保存云草稿”。
4. 点击“清除云草稿”会删除该会话的云草稿。

云草稿保存在 Telegram 服务器，会同步到同一账号的其他客户端。

### 会话文件夹

- 点击“刷新文件夹”读取 Telegram 当前的自定义/共享文件夹。
- 输入文件夹名称。
- 在会话列表中使用 `Ctrl` 或 `Shift` 多选会话。
- 点击“创建文件夹”。

文件夹数量、允许加入的会话类型及共享能力由 Telegram 当前账号权限决定。

## 五、数据与排错

- `settings.dat`：DPAPI 加密的账号配置、规则和通知偏好。
- `outbox.db`：可靠发件箱；正文使用 DPAPI 加密。
- `messages.db`：账号隔离的本地消息搜索索引。
- `logs`：运行日志目录。

如果操作失败，先查看主窗口底部状态和“运行日志”。程序内部空引用等非业务异常会写入异常日志；权限不足、输入为空等正常业务提示只弹窗，不进入异常库。
